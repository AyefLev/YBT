# 教师课程资产全链路工作流 Design

## 目标

把现有分散在课程体系、知识库、教案、习题、班级作业中的能力串成教师可连续使用的主流程：

1. 教师创建课程、章节、课次和知识点。
2. 教师给某个课程节点上传或选择资料。
3. 教师基于课程节点、知识点、资料和自定义提示词生成教案。
4. 教师基于同一课次、教案、知识点和资料生成习题。
5. 习题经过多 AI 审核和自动修订后，教师可手动编辑并保存。
6. 教师把习题沉淀到题库，或发布为班级作业。
7. 所有内容在课程资产树中按层级归类查看。

本轮目标是打通产品体验和轻量接口补齐，不重做现有课程、材料、教案、习题和班级底层数据模型。

## 当前基础

后端已有这些关键能力：

- `Course -> Chapter -> LessonSession -> KnowledgePoint` 课程树。
- `Material`、`Lesson`、`Exercise` 已有 `course_id`、`chapter_id`、`session_id`、`knowledge_point_id`。
- 教案生成会拼入课程树上下文，并支持 `prompt_template`、`output_format`、`web_search_enabled` 预留字段。
- 习题生成支持关联 `lesson_id`，支持多 AI 审核与自动修订。
- 习题可保存到题库草稿。
- 班级作业已有 `exercise_id` 字段，但前端发布表单没有把已保存习题接入。
- 课程页已有资产树展示雏形，但缺少从树节点进入生成、上传、发布的快捷动作。

主要缺口在前端工作流整合、材料选择体验、作业发布体验和左侧工作台布局。

## 用户流程

### 1. 课程资产树作为主入口

课程体系页仍负责创建课程、章节、课次和知识点，但选中课程后，资产树需要承担“主工作台”的作用。

每个层级展示：

- 章节：章节摘要、直属知识点、直属材料、直属教案、直属习题。
- 课次：课次目标、课次知识点、课次材料、课次教案、课次习题。
- 知识点：说明、难度、关联材料、教案、习题。

每个课次和知识点提供快捷操作：

- 上传资料：跳转到知识库页，并带上课程上下文。
- 生成教案：跳转到教案页，并带上课程上下文。
- 生成习题：跳转到习题页，并带上课程上下文。
- 发布作业：跳转到班级作业页，并带上课程上下文。

### 2. 上下文 URL 传递

使用 query 参数传递上下文：

```text
course_id=1
chapter_id=2
session_id=3
knowledge_point_id=4
lesson_id=5
exercise_id=6
```

前端页面挂载时读取这些参数，自动选择课程、章节、课次、知识点、教案或习题。用户仍可手动修改。

这比新建复杂全局状态更稳，也能支持刷新页面和分享链接。

### 3. 材料选择器

教案页和习题页不再只让用户手填材料 ID。新增材料选择区：

- 默认展示当前课程、章节、课次、知识点相关材料。
- 可切换查看所有可用材料，包括个人资料和公共资料。
- 支持多选，生成时仍提交 `material_ids`。
- 保留手填 ID 作为高级兜底入口，但默认隐藏或放在“高级设置”里。

页面内上传资料本轮采用跳转方案：

- 从教案/习题页点击“上传资料”跳到材料页。
- 材料页读取 query 上下文，上传后自动归属到对应课程节点。
- 上传完成后保留返回教案/习题页的入口。

### 4. 教案生成

教案页打开时，如果 query 有课程上下文：

- 自动填充课程、章节、课次、知识点。
- 章节名、课次目标、知识点描述用于生成参数默认值。
- 材料选择器默认勾选当前节点相关资料。

生成时：

- 提交课程树 ID、选中资料、提示词、格式要求和网络检索预留开关。
- 后端继续使用现有 `generate_lesson()` 逻辑。
- 生成结果展示初稿、引用资料、合规状态、多 AI 审核意见和修订稿。
- 教师可采用修订稿或手动编辑后保存。

PPT 生成：

- 教案保存后显示“生成 PPT”按钮。
- 调用现有 `POST /api/presentations/lesson/{lesson_id}/generate`。
- 当前接口可返回任务状态为 queued 或 skipped；UI 明确显示“接口已预留，渲染器未实现时会记录跳过任务”。

### 5. 习题生成

习题页打开时，如果 query 有课程上下文：

- 自动填充课程、章节、课次、知识点。
- 默认筛选同一课次下的教案，优先选择最新教案。
- 默认将知识点名称填入 `knowledge_point`。
- 材料选择器默认展示当前节点相关资料。

生成时：

- 提交课程树 ID、`lesson_id`、选中资料、题型、难度、数量、提示词、格式要求。
- 多 AI 审核和自动修订作为显式开关，默认开启。
- 生成结果展示初稿、审核意见、修订稿、合规状态和引用资料。
- 教师可以手动编辑后保存为习题。
- 保存后支持保存到题库草稿。

### 6. 发布作业

班级页发布作业表单增强：

- 选择已保存习题。
- 默认按当前班级绑定课程过滤习题。
- 如果 query 传入 `course_id/session_id/exercise_id`，自动筛选并预选。
- 选择习题后自动填充作业标题、说明和要求。
- 发布时提交已有 `exercise_id` 字段。

学生端仍按现有作业提交流程处理；本轮不扩展自动批改。

### 7. 左侧工作台和布局优化

左侧工作台支持收缩：

- 宽屏下显示完整导航。
- 点击收缩后只保留品牌标识和导航短标签。
- 收缩状态写入 `localStorage`。
- 移动端仍使用现有顶部/流式布局，不强制固定侧栏。

布局优化原则：

- 课程页：左侧课程列表，右侧详情和资产树，关键操作放到节点附近。
- 教案/习题页：上方是上下文选择，中部是材料与生成参数，下方是生成结果和保存动作。
- 班级页：发布表单增加“关联习题”区域，避免和学生提交/批改列表混在一起。

## 后端设计

优先复用现有接口，只补齐必要读取能力：

1. `GET /api/materials` 已返回课程归属字段，可用于材料选择器。
2. `GET /api/lessons` 已返回课程归属字段，可用于习题页筛选教案。
3. `GET /api/exercises` 已返回课程归属字段，可用于班级页筛选习题。
4. `POST /api/classrooms/{id}/assignments` 已支持 `exercise_id`，但需要校验习题归属和教师权限。

需要补齐的后端行为：

- 创建作业时，如果传入 `exercise_id`，验证当前教师可访问该习题。
- 如果班级绑定了 `course_id`，传入习题的 `course_id` 必须为空或与班级课程一致。
- `AssignmentRead` 可继续只返回 `exercise_id`；前端用本地习题列表补齐标题。

暂不新增真实网络检索后端实现。`web_search_enabled` 本轮仍作为预留字段，UI 标注“离线演示版本不主动联网”。

## 前端设计

新增或修改的前端单位：

- `contextQuery.ts`：解析和生成课程上下文 query。
- `teachingContext.ts`：从课程详情中推导章节、课次、知识点默认文案。
- `MaterialPicker.vue`：多选资料选择器，支持当前上下文和全部资料视图。
- `WorkbenchLayout.vue`：侧栏收缩。
- `CoursesPage.vue`：资产树快捷动作。
- `MaterialsPage.vue`：读取 query 上下文并上传归属。
- `LessonPage.vue`：读取 query、材料选择器、PPT 按钮。
- `ExercisePage.vue`：读取 query、材料选择器、默认教案选择、保存题库入口保留。
- `ClassroomsPage.vue`：选择习题发布作业。

如果为了控制范围，`MaterialPicker` 可以先作为页面内局部代码实现；但推荐抽成组件，因为教案页和习题页会共用。

## 测试策略

后端：

- 增加班级作业关联习题权限测试。
- 增加班级课程和习题课程不一致时拒绝发布的测试。

前端：

- 增加 query 上下文解析工具测试。
- 增加课程资产快捷链接生成测试。
- 增加材料筛选工具测试。
- 保持现有 router、auth、courseStructure 测试通过。

验证命令：

```powershell
cd backend
.\.venv\Scripts\python.exe -m pytest tests\test_classrooms_permissions.py tests\test_courses.py tests\test_lessons.py tests\test_exercises_exports.py -q --basetemp .pytest-tmp

```

全量后端测试如果耗时过长，至少跑本轮相关测试；最终提交前再尝试全量。

## 非目标

本轮不做：

- 真实联网检索和自动构建知识库。
- 真正 PPT 渲染器。
- 图片素材库和视觉模型全链路。
- 自动批改学生作业。
- 大规模数据库迁移系统。

这些能力可以在当前全链路稳定后继续扩展。

## 风险

- 当前本地仓库显示没有提交，但远端已配置。推送前必须先检查远端 `main`，避免覆盖同伴提交。
- 后端全量测试曾出现 AI 服务相关失败，需要在实现前确认同伴提交是否已经修复，或在本轮一并修掉。
- 前端页面较大，改动时要避免把 `LessonPage.vue` 和 `ExercisePage.vue` 继续膨胀到难以维护；材料选择和 query 逻辑优先抽到独立文件。
